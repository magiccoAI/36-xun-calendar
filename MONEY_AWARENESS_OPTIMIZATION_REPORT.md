# 金钱观察模块按日期存储优化报告

## 优化目标
解决金钱观察模块在不同日期间数据混淆的问题，实现每日独立草稿存储和实时自动保存。

## 核心改进

### 1. 按日期存储草稿
- **之前**: 使用固定 key `moneyObservationDraft` 存储所有日期的草稿
- **现在**: 使用 `moneyObservationDraft_${YYYY-MM-DD}` 格式，每个日期独立存储

### 2. 实时自动保存
- **之前**: 只在手动点击"保存草稿"时保存
- **现在**: 用户每次选择/输入都立即自动保存

### 3. 智能状态管理
- **新日期**: 显示空白问卷，专注当日记录
- **已填写日期**: 优先加载正式数据，其次加载草稿
- **跨日期操作**: 自动保存每个日期的进度，数据不混淆

## 技术实现

### MoneyAwarenessModule.js 修改

#### 新增属性
```javascript
this.currentDate = null; // 当前操作的日期
```

#### 新增方法
- `getDraftKey(date)`: 生成日期特定的草稿key
- `loadDraft(date)`: 加载指定日期的草稿
- `clearDraft(date)`: 清除指定日期的草稿
- `hasDraftForDate(date)`: 检查某日期是否有草稿
- `cleanupOldDraft()`: 清理旧格式草稿，实现向后兼容

#### 修改方法
- `onContentClick()`: 添加实时保存
- `onFooterClick()`: 步骤切换时保存
- `goNext()`: 步骤切换时保存
- `setData()`: 支持日期参数

### Modal.js 修改

#### 修改方法
- `initMoneyAwarenessModule()`: 添加日期参数，智能加载正式数据或草稿
- 调用处传递当前日期: `this.currentDateStr`

- ✅ 每次选择选项立即保存

- ✅ 步骤切换时保存进度
